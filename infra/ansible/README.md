# Ansible

Durable provisioning for the development VM.

Yard creates the VM with Lima. Ansible then connects through the SSH config generated by Lima:

```bash
ssh -F ~/.lima/<vm>/ssh.config lima-<vm>
```

Current target:

- official Docker Engine inside the VM;
- Devcontainer CLI in `/opt/devtools`;
- versioned Supabase CLI from GitHub Releases with checksum verification, only when the Supabase adapter is enabled;
- `ubuntu` user in the `docker` group;
- hardened `/proc` with `hidepid=2` when compatible;
- `/home/ubuntu/workspaces` directory.

Runtime secrets do not pass through Ansible.
